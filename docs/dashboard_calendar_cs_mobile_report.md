# 지필드 대시보드/CS/캘린더 구조 변경 및 모바일 리포트 수정 보고서

## 1. 구조 변경 배경
기존 시스템은 리포트 작성, CS 기록, 결석/보강, To-Do, 캘린더, Notion이 한 화면 안에서 함께 움직였지만 기준 데이터가 분산되어 있었다. 특히 To-Do와 수업_활동_기록만으로 일정을 판단하면 결석 후 보강처럼 하나의 사건에 일정이 2개 생겨야 하는 경우를 안정적으로 추적하기 어려웠다.

## 2. 기존 문제
- To-Do는 업무 체크리스트인데 일정 원본처럼 사용되어 캘린더 조회 결과가 누락되거나 잘못 확정될 수 있었다.
- 결석과 보강이 같은 사건으로 섞여 저장되어 결석일과 보강일을 각각 캘린더에 만들기 어려웠다.
- 보강 변경/취소 시 기존 캘린더 이벤트를 찾아 업데이트하거나 삭제할 기준 ID가 부족했다.
- CS 관리 안에 출결 변경, 보강 취소, 퇴원 같은 상태 전이 업무가 섞여 업무 흐름이 불명확했다.
- AI 비서가 To-Do 중심으로 답하면 실제 운영 일정과 다른 답을 낼 수 있었다.

## 3. 변경 방향
- 수업_활동_기록은 문서/리포트/상담 저장 기록으로 유지한다.
- 실무_TODO는 업무 체크리스트로 유지한다.
- 학생DB 관리는 학생 상태 기준표로 유지한다.
- 새 기준표인 일정_마스터를 추가해 실제 일정의 원본으로 삼는다.
- Google Calendar와 Notion은 일정_마스터의 ID, 캘린더이벤트ID, 노션페이지ID를 기준으로 생성/수정/취소한다.
- AI 비서는 바로 실행하지 않고 미리보기와 승인 흐름을 거친다.

## 4. 캘린더/CS 대처
- 결석 등록은 [결석] row와 캘린더 이벤트를 만든다.
- 보강 확정은 별도 [보강확정] row와 캘린더 이벤트를 만든다.
- 같은 사건은 관련그룹ID로 묶어 함께 조회한다.
- 보강 변경은 기존 이벤트를 중복 생성하지 않고 기존 ID를 기준으로 변경하거나 삭제 후 새 이벤트를 만든다.
- 보강 취소는 일정_마스터 상태와 To-Do 기록을 남기고 캘린더 이벤트를 삭제 또는 취소 처리한다.
- 퇴원예정/퇴원확정은 CS 메모가 아니라 학생 상태 전이 업무로 분리한다.

## 5. 모바일 리포트 수정
- 학습 리포트의 과제 확인 영역에서 과정/수행/코멘트를 3열로 고정하던 구조를 변경했다.
- 모바일에서는 코멘트가 과정과 수행 아래에 왼쪽 정렬로 표시되도록 했다.
- CS 안내 메시지 카드에는 min-width:0, minmax(0,1fr), overflow-wrap:anywhere를 적용해 긴 안내문, 계좌번호, 괄호 문구가 화면 밖으로 잘리지 않도록 했다.
- 결석/보강 공지 카드도 긴 사유와 일정이 모바일에서 자연스럽게 줄바꿈되도록 보강했다.

## 6. 검수 포인트
- 모바일에서 과제 확인 코멘트가 오른쪽 좁은 칸에 갇히지 않고 아래로 내려오는지 확인한다.
- 모바일에서 CS 안내 메시지 본문이 좌우로 잘리지 않고 줄바꿈되는지 확인한다.
- 결석/보강 공지에서 긴 사유와 보강 시간이 화면 밖으로 밀리지 않는지 확인한다.
- 일정 관련 AI 조회는 To-Do가 아니라 일정_마스터 기준으로 답하는지 확인한다.
